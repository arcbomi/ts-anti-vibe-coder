# AI Question Rules

## Required Rules

- Questions must be English only.
- Questions must use A/B/C/D options.
- Each question must have only one correct answer.
- Each question must include an explanation.
- Each question must include `source_file_path`.
- Questions should test real code understanding.
- Questions must avoid shallow README or metadata facts.
- The AI must generate exactly 20 questions for each repository exam.
- There is no admin review step; generated questions are used directly.

## Good Question Topics

Questions should focus on functional understanding, such as:

- How the program starts.
- Which function registers HTTP routes.
- How `mux`, `net/http`, or another router is used.
- What happens when an endpoint is called.
- Which service function is called by a handler.
- How request data is parsed.
- How response data is returned.
- How errors are handled.
- What would break if middleware is removed.
- Where a new feature should be added.
- How frontend state moves through hooks, store, and components.

## Bad Question Topics

Avoid shallow questions such as:

- What is the repository name?
- What language is used?
- What is the README title?
- What package version is used?
- What is the exact variable name?

## AI Output Format

```json
{
  "questions": [
    {
      "question": "How are HTTP routes connected to handlers in this repository?",
      "options": {
        "A": "Routes are automatically generated from file names.",
        "B": "Routes are manually registered using mux.HandleFunc or similar router setup.",
        "C": "Routes are loaded from the database.",
        "D": "Routes are generated by the frontend."
      },
      "correct_option": "B",
      "explanation": "The route setup file manually connects URL paths to handler functions.",
      "difficulty": "medium",
      "source_file_path": "internal/server/router.go"
    }
  ]
}
```

## Validation Expectations

Before saving questions, the backend should validate that:

- There are exactly 20 questions.
- Every question is written in English.
- Every question has options `A`, `B`, `C`, and `D`.
- `correct_option` is one of `A`, `B`, `C`, or `D`.
- Every question has a non-empty explanation.
- Every question has a non-empty `source_file_path`.
